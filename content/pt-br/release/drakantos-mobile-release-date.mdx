### Considerações sobre Adaptabilidade Mobile

Embora nenhuma versão mobile tenha sido confirmada, um port bem-sucedido precisaria abordar várias áreas-chave:

| Recurso           | Implementação para PC                              | Desafios / Soluções de Adaptação para Mobile                 |
| :---------------- | :------------------------------------------------- | :------------------------------------------------------- |
| **Controles**     | Teclado/mouse, suporte a controle recomendado      | **Desafio**: Precisão da tela sensível ao toque; **Solução**: Joysticks virtuais, botões de habilidade simplificados |
| **UI/UX**         | Interface limpa e moderna, navegação fácil         | **Desafio**: Restrições de tamanho de tela; **Solução**: UI escalável, menus sensíveis ao contexto |
| **Desempenho**    | Mínimo: Intel Core i3, 4GB RAM, 1GB VRAM           | **Desafio**: Fragmentação de dispositivos, consumo de bateria; **Solução**: Gráficos otimizados, opções de configurações mais baixas |
| **Multiplayer**   | PvP Online, Masmorras Cooperativas                 | **Desafio**: Estabilidade da rede em mobile; **Solução**: Infraestrutura de servidor robusta, servidores regionais |
| **Armazenamento** | Mínimo: 1 GB de espaço disponível                  | **Desafio**: Limites de armazenamento do dispositivo; **Solução**: Downloads modulares, tamanhos de ativos otimizados |

<Callout type="warning" title="Potenciais Desafios Mobile">
Adaptar um MMORPG como Drakantos para mobile envolve desafios significativos, incluindo otimizar controles para telas sensíveis ao toque, escalar a interface do usuário para telas menores e garantir um desempenho estável em uma ampla gama de dispositivos mobile.
</Callout>

### Abordando Preocupações dos Jogadores

Durante vários playtests, os jogadores levantaram vários pontos em relação à jogabilidade. Os desenvolvedores abordaram ativamente essas questões, indicando uma abordagem responsiva que seria vital para qualquer port mobile.

<Steps>
  <Step num="1" title="Fluidez do Combate">
    O feedback inicial notou travas de animação e movimento desajeitado. Os desenvolvedores reconheceram isso, prometendo correções para testes posteriores para melhorar a sensação geral do combate. Isso é crítico tanto para PC quanto para potenciais experiências mobile.
  </Step>
  <Step num="2" title="Mecânicas de Ataque Básico">
    Os jogadores não gostaram do tempo de recarga de 2 segundos nos ataques básicos, especialmente para personagens de longo alcance, dificultando o kiting. A equipe está considerando mudanças para permitir ataques básicos repetíveis, melhorando o fluxo do combate.
  </Step>
  <Step num="3" title="Ritmo de Progressão">
    Os desbloqueios iniciais de habilidades foram considerados muito lentos (níveis 5, 10, 15). Os desenvolvedores ajustaram isso para os níveis 4, 8 e 12, tornando a progressão inicial do jogo mais recompensadora.
  </Step>
  <Step num="4" title="Bloqueio de Corpo por NPCs">
    A questão de NPCs e membros do grupo bloqueando o movimento do jogador foi identificada. Essa preocupação foi reconhecida e está programada para uma correção, o que melhoraria muito a navegação e o posicionamento de combate.
  </Step>
</Steps>

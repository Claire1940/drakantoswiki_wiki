<Section icon={Swords} title="コスメティックなパーソナライゼーションオプション">
Drakantosでの装備の進行は、視覚的な変更よりも主にパワーと統計に焦点を当てていますが、プレイヤーはヒーローの外見をカスタマイズするための手段をまだ持っています。これらのオプションは主にコスメティックであり、個人の表現を可能にします。

### 利用可能なコスメティックカスタマイズ要素

| カスタマイズの側面 | 説明 | 外見への影響 |
| :------------------- | :---------- | :------------------- |
| **スキン**            | ヒーローのユニークな衣装と別の形態。 | ヒーローの外見を劇的に変えることができます（例：モハザスの「生きている」バージョン）。 |
| **カラーパレット**   | 衣装やアクセサリーの配色を変更するための染料システム。 | 広範な個人的な色の選択を可能にします。 |
| **ヘアスタイル**       | 適用可能なヒーローのための異なるヘアオプション。 | 頭飾りや髪に多様性を提供します。 |
| **アーマー/アクセサリーパーツ** | ヘルメット、ケープ、ひげなどのコスメティックなパーツ。 | キャラクターモデルにユニークな魅力を加えます。 |
| **マウント＆ペット**    | キャラクターに追従したり、キャラクターを乗せたりする仲間や乗り物。 | 視覚的な存在感を高め、個性を加えます。 |
| **エモート**           | 感情を表現するための特別なアニメーション。 | キャラクターのインタラクションと表現に影響します。 |

<Callout type="info" title="視覚的な進行に関する注意">
多くのMMORPGとは異なり、Drakantosにおけるキャラクターの視覚的な外見は、着用している装備と直接結びついていません。アーマーや武器をアップグレードするとステータスは向上しますが、特定のコスメティックスキンや染料を適用しない限り、ヒーローの外見は変わりません。
</Callout>

</Section>

<Section icon={Target} title="スキルベースのカスタマイズ：オーブ、アーティファクト、トロフィー">
コスメティック以外にも、Drakantosはヒーローのアビリティとプレイスタイルをカスタマイズするための堅牢なシステムを提供しています。この「ビルド作成」には、オーブ、アーティファクト、トロフィーの組み合わせが含まれており、プレイヤーはヒーローのパフォーマンスを好みに合わせて調整できます。

### スキルカスタマイズの仕組み

| コンポーネント | 機能 | ゲームプレイへの影響 |
| :--------- | :------------------------------------------ | :---------------------------------------------------- |
| **オーブ**   | ヒーローの基本スキルを変更します。 | アビリティを強化したり、効果を変更したり、新しい特性を追加したりします。 |
| **アーティファクト** | ヒーローのツールキットに2つのアクティブスキルを追加します。 | 新しい戦闘オプションと戦略的な深さを導入します。 |
| **トロフィー** | 2つのパッシブスキルを付与します。 | 常にバフを提供したり、ユニークな状況に応じた利点を提供したりします。 |

<Callout type="success" title="ビルドを深める">
オーブ、アーティファクト、トロフィーのさまざまな組み合わせを試すことは、自分のプレイスタイルに合ったビルドを見つけるために不可欠です。このシステムは、伝統的なキャラクター作成がなくても、かなりの戦略的な深さを可能にします。
</Callout>

ゲームには事前に定義されたヒーローが登場しますが、スキルカスタマイズシステムにより、同じヒーローのプレイヤーが必ずしも同じようにプレイするわけではないことが保証されます。たとえば、オーブは単一ターゲット攻撃をAoEに変更したり、アーティファクトは強力な新しい防御アビリティを付与したりすることができます。これにより、ダイナミックでパーソナライズされた戦闘体験が可能になります。

</Section>

<Section icon={Shield} title="ヒーローベースのアプローチ vs. 伝統的なキャラクター作成">
伝統的なキャラクター作成の代わりにヒーローベースのシステムを使用するという決定は、Drakantosにおける意図的なデザイン選択です。このアプローチには、深いカスタマイズを求めるプレイヤーにとっての利点と欠点の両方があります。

### ヒーローベース vs. 伝統的な作成

| 特徴 | ヒーローベースシステム (Drakantos) | 伝統的なキャラクター作成 (典型的なMMORPG) |
| :------------------ | :------------------------------------------- | :----------------------------------------------- |
| **最初の選択** | 事前にデザインされたヒーローのリストから選択します。 | 外見をデザインし、種族、クラス、性別を選択します。 |
| **伝承の統合** | ヒーローの背景と物語への深い繋がり。 | プレイヤーがキャラクターの物語を定義します。 |
| **視覚的な多様性** | スキン、染料、コスメティックパーツに依存します。 | 装備が外見をしばしば変更します。広範なスライダー。 |
| **スキルカスタマイズ** | オーブ、アーティファクト、トロフィーがアビリティを変更します。 | タレントツリー、スキルポイント、装備のステータス。 |
| **性別/種族ロック** | ヒーローは固定の性別/種族を持ちます。スキンが代替を提供する場合があります。 | プレイヤーが性別/種族を自由に選択します。 |

<Callout type="warning" title="プレイヤーの期待">
MMORPGで広範なキャラクター作成に慣れているプレイヤーは、Drakantosのヒーローベースシステムを最初は制限的に感じるかもしれません。しかし、焦点はヒーローのユニークなツールキットを習得し、そのアビリティをカスタマイズすることに移ります。
</Callout>

開発者は、一部のプレイヤーがゼロからアバターを作成する自由を好むかもしれないことを認識しています。しかし、彼らのビジョンは、各ヒーローの物語の深さと明確なアイデンティティを優先しています。これにより、ゲームの世界やクエストに直接織り込まれた、より複雑な背景や個性を生み出すことができます。

</Section>

<Section icon={Trophy} title="Drakantosヒーロービルドの最適化">
Drakantosでの体験を真にパーソナライズするには、コスメティックな選択とスキル変更の相乗効果に焦点を当ててください。最適化されたビルドは、美学と戦闘効果の両方を考慮します。

<Steps>
  <Step num="1" title="ヒーローの核となる強みを理解する">
    各ヒーローにはユニークなプレイスタイルがあります。カスタマイズする前に、彼らの基本アビリティ、強み、弱みを徹底的に理解してください。これがオーブとアーティファクトの選択を導きます。
  </Step>
  <Step num="2" title="オーブとアーティファクトを試す">
    スキルのために異なるオーブ構成を試すことを恐れないでください。アーティファクトはまったく新しいアクティブアビリティを導入するため、ヒーローのキットを補完したり、弱点をカバーしたりするものがどれであるかをテストしてください。
  </Step>
  <Step num="3" title="パッシブ効果のためにトロフィーを選択する">
    トロフィーは、パフォーマンスに大きく影響するパッシブボーナスを提供します。主要なステータスを強化したり、ユーティリティを提供したり、アクティブスキルと相乗効果を発揮したりするトロフィーを選択してください。
  </Step>
  <Step num="4" title="スキンと染料で外見をカスタマイズする">
    ビルドが設定されたら、ヒーローの外見をパーソナライズしてください。コスメティックスキンを使用して全体的な変更を行うか、カラーパレットとアクセサリーパーツを使用して外見を微調整してください。
  </Step>
  <Step num="5" title="ビルドを見直し、適応させる">
    メタが変化したり、新しいコンテンツが新しい課題をもたらしたりする可能性があります。ビルドの有効性を定期的に見直し、必要に応じてオーブ、アーティファクト、トロフィーを適応させる準備をしてください。
  </Step>
</Steps>
